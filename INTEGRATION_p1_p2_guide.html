Pollen DEK Integration Guide

Phase 1 (Rollback Loop) & Phase 2 (State Consistency & Supervisor)

System Architecture Overview

The Pollen DEK Supervisor handles high-availability background execution, policy proxying (MCP), and resilient binary updates across Windows and Linux. The updates include strict health-gating and crash-loop automatic rollbacks.

The Full Crash-Loop Rollback Flow

When an over-the-air binary update is applied, the system uses a state machine to guarantee safety. If the new binary is corrupted, panics, or cannot reach the cloud, it will revert itself.

1. Update Staged: The old binary downloads the new binary, swaps it to the target location, creates an update_pending.json marker, and exits.

2. Service Restart: Systemd or Windows Service Manager restarts the process (which is now the new binary).

3. Probation Boot: The new binary detects the marker. It starts all modules (IPC, Telemetry, Proxy) and delays committing the update.

4. Health Gating: probation.rs pings the local IPC (ipc_client.rs), checks mTLS with the cloud, and verifies bundle parsing.

5a. Commit (Success): If 3 consecutive health checks pass, the marker and .bak are deleted. Probation ends.

5b. Exit (Failure): If health checks fail within 60s, the process gracefully restores the .bak via self_replace and exits, causing the Service Manager to restart into the old binary.

5c. Crash-Loop (Panic): If the binary panics immediately (e.g., segfault), Systemd watchdog triggers. After 5 restarts in 60s (StartLimitBurst), Systemd invokes dek-rollback.service via OnFailure=.

6. dekctl rollback: The oneshot service runs dekctl rollback, reads the marker, restores the binary, and restarts pollen-dek.service.

Integration Steps Completed

1
Real IPC Health Client

Created dek-core/src/ipc_client.rs using LinesCodec to send IpcRequest::HealthCheck and parse HEALTHY status. Replaces the placeholder TCP connect.

2
Emergency Rollback CLI

Added dekctl rollback logic inside dek-cli/src/service/rollback.rs to safely restore the .bak executable using OS-specific extensions (std::env::consts::EXE_EXTENSION).

3
Updater Marker Precision

Updated dek-core/src/updater.rs to include target_path in the marker JSON so the rollback tool knows exactly where to restore.

4
Systemd Units

Created packaging/systemd/units.service detailing the Type=notify, WatchdogSec, and OnFailure=dek-rollback.service properties.

5
Proxy Policy Snapshot

Migrated dek-mcp-proxy state to a lock-free ArcSwap<PolicySnapshot>. This binds DekMetadata, Verifier, and PolicyRouter together, ensuring atomic hot-reloads and eliminating race conditions.

6
Supervisor Extraction

Split the monolithic 839-line main.rs into a structured Supervisor holding a JoinSet and CancellationToken. Network logic moved to ipc_server.rs, bundle_loop.rs, and metrics_push.rs.
